# v4.9 codesop 减负 spec

**日期**：2026-07-05
**主题**：codesop 减负（simple 出口 + 测试术语锚/措辞锁区分 + 去重降噪）
**作者**：claw + Claude（brainstorming）+ Workflow（6-finder 调研）+ codex（独立审）

## 背景

codesop 定位是「大模型的指引/锦上添花」，不是「限制」。Workflow 6-finder 调研 + codex 独立审 + Claude 对照，确认整体锦上添花为主，但有 3 处可减负（按 ROI 综合排序）：

1. simple/trivial 任务过重（缺跳 codex 出口）—— 运行时限制，直接影响大模型发挥
2. 测试锁措辞不锁行为（91 needle 卡 patch 改写）—— 演进限制
3. 多处重复声明（信号稀释 + SKILL.md 膨胀）

核心范式（spec-as-goal / 三 gate / 外部锚点 AND / diff 守护）不动。

## 目标

按综合排序减负，让 simple 任务少跑重流程、测试不再卡措辞、SKILL 信号密度提升。

## 范围

只改 codesop repo：`SKILL.md` / `config/codesop-router.md` / `patches/superpowers/brainstorming-SKILL.md` / `tests/` / `templates/system/AGENTS.md`（后者由 `setup --host claude` symlink 同步到 `~/.claude/CLAUDE.md`）。

**不含**：`~/.claude/CLAUDE.md:36` 全局 `1% skill` 规则（上游 superpowers 纪律，影响所有 skill，建议用户单独评估）。

### Section 2 — 测试术语锚 vs 措辞锁（🥈）

#### 2.1 术语锚 golden-lock
- **完成条件**：5 套行为测试（spec-as-goal / adversarial-review / first-principles / goal-collaboration / spec-gate-visualize）的核心契约术语保留精确锁，每术语至少 1 条 `grep -Fq` 断言。术语清单（≥8）：`ready/approved` · `三件` · `不静默改走普通执行` · `blocking/advisory` · `外部锚点 AND` · `diff 守护` · `spec-gate 唯一硬审` · `pre-/goal preparation segment`
- **边界**：术语锚清单不缩窄——这 8 个是 codesop 核心契约，防 AI 偷偷改 spec 关键概念；漏任何一个 = 防回归弱化
- **风险分级**：**medium**（术语锚漏 = 契约可被无声篡改）

#### 2.2 措辞锁语义化
- **完成条件**：5 套测试的展示文案 / 标题 / 解释句 needle 改为 section 标题 + 1-2 关键词；91 needle 减到 ≤50（其中 ≥8 术语锚 + 其余 section 锚）。`wc -l` 每套 test 的 assert 行数减 ≥30%
- **边界**：每个机制至少保留 1 个语义锚（section 标题或关键判定句）——不能全部释放；释放后该机制的 section 删/关键判定改仍报红
- **风险分级**：**medium**（释放过多 = 机制无守护）

#### 2.3 补 dogfood dispatch 实测
- **完成条件**：`tests/` 加 dogfood 脚本（实测 brainstorming/verification spec-gate 流程一次完整跑通：dispatch subagent → serve → ready/approved 状态机），run_all 含；run_all 总数 ≥20
- **边界**：dogfood 不能替代术语锚（术语锚仍 grep 精确锁）；dogfood 是措辞锁的语义补充
- **风险分级**：**medium**
