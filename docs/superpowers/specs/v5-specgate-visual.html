1要解决的问题

codesop 是给 AI 编码助手套上 SOP 纪律的操作系统（装 10 个 skill、四条铁律、路由表、pipeline 可视化）。 但它带着弱模型时代的包袱：所有任务一律走全套仪式（spec/对齐块/HTML 全上）、 Claude 专属机制（/goal、SessionStart、patch）混进了核心逻辑、 patch 静默失效了 9 天没人知道（6.1.1 升级后 4 个核心补丁被跳过）、 还在假装 Codex / OpenCode 和 Claude 能力对等。 仪式一刀切浪费强模型，假对等掩盖真实差距。

2v5 实际改变什么

v5 后 codesop 变成按风险比例做事的工具，三层效果： ①流程分档——低风险小改动走 minimal（直接改+验证），鉴权/迁移/部署才走 governed（spec+人审+证据+回滚），中间档 standard； ②核心解耦 Claude——拆出 core/（宿主中立）和 claude-adapter/，七类核心纪律跨宿主一致，Claude 编排能力归 Claude； ③事实完整性——runtime 版本/manifest hash/capability 状态每会话自检，报「覆盖完整」前必须真的完整（patch 失效不再静默 9 天）。

3为什么这样改

核心立场一句话：帮模型做对，而不是防模型做错。 强模型时代，硬防只会挡掉正确判断——该信任模型的判断力 + 宿主原生的审批/sandbox， codesop 只负责给出清晰的 floor（流程强度要求）和不变量。 不再自造执行点 / deny hook 冒充安全边界（那是宿主的活）。 实务上单人维护：选覆盖最优的 patch 模式 + 轻量测试（表驱动+dogfood），砍掉完整评测平台这种「第二项目」。

4不改变什么

四条铁律原样保留：先设计再编码 / 先失败再生产 / 无根因不修 bug / 无证据不完工。 七类不变量（治理内核）不动、不掺 skill 名 / /goal / 固定章节。 证据后声明不变——没跑测试看到结果就不能说「已完成」。 5 个 superpowers patch 是命脉（v4.9.1 逐个 md5 验过全必要），模式保持，只加 fingerprint 门禁防吞上游修复。 governed 人审不取消——破坏性操作必须真停下请求批准。

③ Rubric 五项判定（§8.5）

1
完成条件齐吗？（R1-R8 三件：完成条件 + 边界 + 风险分级）
每个 R 都有三件。例 R2 Profile 判定：完成条件=表驱动 12-20 场景全过；边界=ambiguity=high 不进 minimal；风险=high。 Phase 0/1/2 各有独立退出条件（可判定）。 理由：八条 R 逐条拆得开，阶段退出条件写成「测试必红/必过」而非「基本完成」。
满足
2
不空泛吗？（完成条件可机械验证，非主观）
R1=golden test + 尺寸上限；R2=12-20 场景全过 + 高风险误入 minimal=0；R3=minimal 不产 spec/HTML（dogfood 看）；R4=core/ grep 不含禁词； R5=governed 真停下请求批准（dogfood）；R7=跨 minor + minor 内 commit 变测试必过；R8=退出可判定。 理由：判定锚点是 grep/dogfood/测试，不是「感觉对了」。
满足
3
伪机器验证吗？（grep 守卫/测试/dogfood 是真机械锚点）
grep 守卫（R4 core/ 不含 /goal/installed_plugins.json/~/.claude/SessionStart）是真 grep； fingerprint 门禁测试（R7 模拟 commit 变→暂停）是真测试； dogfood（R3/R5/R8）靠人执行但文本契约可断言（v4.9.0 已立 dogfood 文本契约模式）。 顾虑：dogfood 仍含人执行环节，非纯机械——但已用文本契约把「人审」锚定到可断言文本，是当前合理上限。
满足（dogfood 半机械）
4
边界遗漏吗？（anti-Goodhart 边界段）
spec §3 设计原则 + §10 非目标明确反 Goodhart：不靠 Prompt 堆叠当护城河 / floor 是 best-effort 非安全边界 / adapter 不弱化宿主原生审批 / 不取消 governed 人审 / 不自造 deny hook 冒充安全边界。 理由：明确划了「codesop 管什么、不管什么」，防 profile/floor 被当成安全门禁误用。
满足
5
风险错分吗？（R1-R8 风险分级合理）
r3 把 R1（改全局契约）和 R5（改人审语义 + 涉宿主审批）从 r2 的 medium 修正为 high——合理； R7 patch 门禁=high（吞上游修复是真失效）合理；R6 事实完整性=medium、R8 评测=medium 合理（行为观察，非契约变更）。 理由：改全局契约/人审语义的 R 标 high，纯观察/汇总的 R 标 medium，分级有据。
满足
