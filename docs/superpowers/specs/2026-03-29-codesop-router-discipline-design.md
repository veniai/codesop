# Design: codesop 路由卡纪律层

## Problem Statement

AI Agent 在长对话中跳过该用的 skill（完全跳过、中途放弃、或用错 skill）。根本原因是上下文稀释：SessionStart 注入的 `using-superpowers` 纪律层随对话轮次增长权重衰减，AI "忘记"要检查 skill。

## Constraints

- 不修改 superpowers 插件文件，插件更新会覆盖修改
- 不替代 `using-superpowers`，而是补充增强
- 路由卡内容必须简洁（30行以内），长内容本身更容易被稀释
- 与现有 `using-superpowers` 的 SessionStart hook 共存

## Premises

1. 单靠文本指令无法抗稀释，需要利用多层级锚点的结构性冗余
2. 任务对齐（强制 AI 复述计划 + 列出将用的 skill）能部分缓解稀释：输出即承诺、复述即重读、用户可见。但该机制本身也受稀释影响，需配合用户手动补救路径
3. 简洁的 routing table 比完整的 pipeline 文档在长对话中更持久

## Architecture: Three-Layer Redundancy

```
Layer 1: SessionStart Hook (路由卡)
  位置: ~/.claude/settings.json hooks → cat ~/.claude/codesop-router.md
  内容: ~30行精简路由表 + 任务对齐规则
  权重: 每次对话开头注入，和 using-superpowers 并列

Layer 2: Global CLAUDE.md
  位置: ~/.claude/CLAUDE.md
  内容: ~10行核心纪律声明 + 指向 /codesop
  权重: 系统提示级别，稳定

Layer 3: /codesop Command
  位置: ~/.claude/commands/codesop.md
  内容: 完整 pipeline + 任务对齐执行
  权重: Skill tool 注入，最高权重上下文
```

冗余逻辑：三层不是做同一件事，而是各有侧重。Layer 1 负责长对话提醒，Layer 2 负责系统级约束，Layer 3 负责用户触发时的完整执行。任何一层被稀释，其他层仍然能介入。

## Task Alignment Mechanism

核心机制：**强制 AI 在动手前输出结构化对齐块**。

```
任务对齐块 (Task Alignment Block)
─────────────────────────────────
每次收到新任务，必须先输出：

🎯 任务对齐
- 理解: [用自己的话复述要做什么]
- 阶段: [Pipeline Stage N: 名称]
- 必用 Skill: [列出将调用的 skill]
- 跳过及原因: [如跳过某阶段，说明为什么]
─────────────────────────────────
```

### Why It Works

1. **输出即承诺** — AI 写出来的计划比系统提示中的规则更难违背。在近期上下文中明确说了"我会用 brainstorming"，后续跳过产生上下文矛盾
2. **复述 = 重新加载** — 写"必用 Skill: brainstorming"本身是对路由表的一次重读
3. **用户可见** — 对齐块对用户可见，用户可以直接指出"你跳过了 brainstorming"

### Trigger Conditions

不是每句话都触发，只在以下节点：
- 收到新功能/新任务请求
- 从一个 pipeline 阶段进入下一个
- 用户说"开始做"/"执行"/"修这个 bug"

## Router Card Content

SessionStart hook 注入的完整路由卡：

> **设计原则：路由卡是 pipeline 的索引，不是简化版。** 所有必走 skill 必须完整列出，
> 与 `commands/codesop.md` 保持严格一致。唯一真相源是 `commands/codesop.md`，
> 路由卡由 `config/codesop-router.md` 在 setup 时从 codesop.md 提取生成。

```markdown
## codesop 路由卡

新任务必须先输出任务对齐块（理解 + 阶段 + Skill）。
完整 pipeline 定义见 /codesop。

### 必走路径（不可跳过）
| 用户信号 | Pipeline 阶段 | 必走 Skill 序列 |
|---------|--------------|----------------|
| 做功能 / 加东西 / 重构 | 探索→计划→执行→验证→review | brainstorming → writing-plans → autoplan → worktrees → subagent-driven-dev → TDD → verification → review |
| 修 bug / 测试挂了 | 调试→验证→review | systematic-debugging → verification → review |
| 做完了 / 修好了 | 验证→review | verification → qa(web) → review |
| 发布 / ship | 发布→清理 | ship → document-release |

### 可选路径
| 用户信号 | Skill |
|---------|-------|
| 方向不明确 | office-hours |
| 测一下 (web) | qa |
| 小心点 | careful |
| 只改这个目录 | freeze |

### 铁律
- 跳过必走 Skill = 先输出对齐块说明原因
- 不确定 → 先调用 /codesop
```

**漂移防控：** `config/codesop-router.md` 由 `setup` 从 `commands/codesop.md` 的 Development Pipeline 章节自动提取生成。不手写路由卡。如果 codesop.md 的 pipeline 改了，setup 同步时路由卡自动更新。

### codesop-router.md 生命周期

源文件在 repo 的 `config/codesop-router.md`，setup 同步到 `~/.claude/codesop-router.md`。
这样路由卡受版本控制，`codesop update` 时自动拉取更新。

### settings.json Change

setup 自动配置，不要求用户手改。新增 `configure_hooks()` 函数：

Claude Code hooks 的正确 schema：
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "cat ~/.claude/codesop-router.md"
          }
        ]
      }
    ]
  }
}
```

实现：
```bash
configure_hooks() {
  local settings="$HOME/.claude/settings.json"

  # 确保 settings.json 存在且是有效 JSON
  if [ ! -f "$settings" ]; then
    mkdir -p "$(dirname "$settings")"
    echo '{}' > "$settings"
  fi

  if ! command -v jq >/dev/null 2>&1; then
    echo "  ⚠ jq not found — manual step required:"
    echo "    Add to ~/.claude/settings.json:"
    echo '    "hooks": {"SessionStart": [{"matcher":"","hooks":[{"type":"command","command":"cat ~/.claude/codesop-router.md"}]}]}'
    return 0
  fi

  # 检查是否已存在 codesop-router hook（幂等）
  local existing
  existing=$(jq '.hooks.SessionStart // [] | [.[] | select(.hooks[]?.command == "cat ~/.claude/codesop-router.md")] | length' "$settings" 2>/dev/null || echo "0")
  if [ "$existing" != "0" ]; then
    echo "  ✓ codesop-router hook already configured"
    return 0
  fi

  # 合并 hook，保留已有 hooks
  jq '.hooks.SessionStart += [{"matcher":"","hooks":[{"type":"command","command":"cat ~/.claude/codesop-router.md"}]}]' \
    "$settings" > "$settings.tmp" && mv "$settings.tmp" "$settings"
}
```

## Success Criteria

- 新任务触发任务对齐块输出率 > 90%（目测统计）
- 必用 skill 跳过率从当前 ~40% 降到 < 10%
- 路由卡在 20+ 轮对话后仍能被 AI 引用
- superpowers 插件更新不破坏任何增强内容
- `codesop setup --host claude` 一条命令完成所有配置（包括 hooks）
