## Task 1: tests/run_all.sh 统一测试入口

**文件**: `tests/run_all.sh`（新建）

创建统一测试运行器，顺序执行所有测试套件：
- 输出每个套件的 PASS/FAIL 状态
- 最后汇总：X/Y passed
- 任一套件失败则整体 exit 1
- 支持 `bash tests/run_all.sh` 一键运行

套件执行顺序：
1. codesop-router.sh
2. detect-environment.sh
3. setup.sh
4. codesop-init.sh
5. codesop-init-interview.sh
6. codesop-symlink.sh
7. codesop-e2e.sh

## Task 2: PRD 模板去重

**文件**: `lib/templates.sh`（修改）, `templates/project/PRD.md`（修改）

当前问题：`templates.sh` 的 `write_prd_template()` 内联了 ~200 行 PRD 模板，与 `templates/project/PRD.md` 是两份独立维护的正文。

方案：`write_prd_template()` 改为读取外部模板文件 `templates/project/PRD.md`，用 sed 做变量替换（`{PROJECT_NAME}` → 实际值，`{DATE}` → 实际日期），然后写入目标文件。内联模板删除。

外部模板 `templates/project/PRD.md` 需补齐当前内联版独有的章节（领域实体、技术规范模板内容），成为唯一真相源。

## Task 3: init-interview.sh Python 注入改 jq

**文件**: `lib/init-interview.sh`（修改，~行 398-425）

当前问题：`ensure_new_init_env()` 用 python3 做嵌套 JSON 写入（`$settings_file` 直接拼入 python 字符串）。

方案：改用 jq，和 setup 脚本保持一致。逻辑：
```bash
jq '.env = (.env // {} | . + {"CLAUDE_CODE_NEW_INIT": "1"})' "$settings_file" > "$settings_file.tmp" \
  && mv "$settings_file.tmp" "$settings_file"
```
保留 python3 作为 jq 不可用时的 fallback（但用安全的临时文件方式传递路径）。

## Task 4: GitHub Actions CI + shellcheck

**文件**: `.github/workflows/test.yml`（新建）, `.shellcheckrc`（新建）

CI workflow：
- 触发：push 到 main、PR 到 main
- 步骤：checkout → 安装 jq/shellcheck → run shellcheck → run tests/run_all.sh
- shellcheck 扫描范围：codesop 入口 + lib/*.sh + setup（排除 tests/ 和 templates/）

`.shellcheckrc`：
- `source-path=lib`（让 shellcheck 找到 source 的模块）
- 排除特定 warning（如 SC2317 对 trap 函数的误报）
