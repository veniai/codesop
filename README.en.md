# codesop

**AI Coding Standard Operating Procedure**

A skill-first operating system for AI-assisted coding workflows. The current core keeps one main flow, `/codesop`, plus two mechanical commands: `init` and `update`.

[![Version](https://img.shields.io/badge/version-3.4.0-blue.svg)](VERSION)

## What is this?

A cross-tool AI coding workflow OS for Claude Code, Codex, OpenCode, and other AI coding assistants.

Core value: give AI assistants unified workflow discipline in any project — knowing which skill to use, what order to execute, and when to stop and verify.

## Install

```bash
git clone https://github.com/veniai/codesop.git ~/codesop
cd ~/codesop && bash install.sh
```

Make sure `~/.local/bin` is on your `PATH`.

## What gets installed?

| Component | Target | Purpose |
|-----------|--------|---------|
| Router card | `~/.claude/codesop-router.md` | SessionStart hook discipline table |
| Slash commands | `~/.claude/commands/` | `/codesop` + `/codesop-init` + `/codesop-update` |
| System AGENTS.md | `~/.claude/CLAUDE.md` → `templates/system/AGENTS.md` | Global AI contract + skill discipline |
| Skill runtime | `~/.claude/skills/codesop/` | Skill file runtime |
| CLI | `~/.local/bin/codesop` | Command-line tool |

## Usage

### Initialize a new project

```bash
/codesop init .
```

### Open the workbench

```bash
/codesop
```

`/codesop` outputs a workbench summary and next-step recommendation. The last line is a natural-language workflow instruction (1-3 skills chained), press Enter to confirm.

### Pipeline-to-todo

`/codesop` converts the recommended chain into a TaskCreate task list (☐/☑ visual progress), preventing AI from forgetting intermediate steps. Re-invoking `/codesop` detects stale chains and re-routes.

### Update

```bash
codesop update
# or
/codesop-update
```

## What does `/codesop init` do?

1. Scan project: detect language, shape, framework
2. Detect environment: Claude Code / Codex / OpenCode / superpowers
3. Generate project files:
   - `AGENTS.md` → `@CLAUDE.md` (lightweight reference)
   - `PRD.md` → living document (product spec + progress + work log)
   - `README.md` → install/run/test commands (if missing)
4. CLAUDE.md is generated by Claude Code's `/init`, codesop doesn't overwrite

## Document Gate

- After any `/codesop`-routed implementation task, decide whether `CLAUDE.md`, `PRD.md`, `README.md` need updates
- If any document needs updating, prefer `claude-md-management`
- `AGENTS.md` stays a thin wrapper to `@CLAUDE.md`
- `CHANGELOG.md` is not part of the default gate

## Versioning

- `VERSION` is the single source of truth for releases
- `skill.json` and `PRD.md` versions must match `VERSION`
- `CHANGELOG.md` uses `Unreleased` at top, renamed to version at ship time
- git tags created at ship only, e.g. `v3.0.1`

## Product Contract

- One main flow: `/codesop`
- Two mechanical commands: `codesop init`, `codesop update`
- Primarily designed and tested for Claude Code; Codex / OpenCode partially supported

## Workflow Scenarios

The routing table (`config/codesop-router.md`) is the single source of truth for chain assembly. Typical scenarios:

| Scenario | Chain |
|----------|-------|
| New feature | brainstorming → codex:rescue → writing-plans → subagent-dev → ☆simplifier → verification → ☆claude-md → finishing → code-review → codex:rescue → receiving-code-review |
| Bug fix | systematic-debugging → verification → ☆claude-md → finishing |
| Small change | subagent-dev → ☆simplifier → verification → finishing |
| Review feedback | receiving-code-review → fix → verification → reply |

☆ = only if plugin available. Full rules in routing table.

## Dependencies

codesop orchestrates these skill ecosystems:

- **[superpowers](https://github.com/obra/superpowers)** — brainstorming, writing-plans, TDD, systematic-debugging, subagent-driven-dev, verification-before-completion, receiving-code-review
- **code-review** — PR review: 5 parallel agents + confidence scoring
- **codex** — codex:rescue dual-AI review (mandatory in design + code review phases)
- **claude-md-management** — document drift check (mandatory after verification, before submission)
- **code-simplifier** — code polish (after development, before verification)

Install:
- superpowers: `/plugin install superpowers` (Claude Code)
- code-review: `/plugin install code-review`
- codex: `/plugin marketplace add openai/codex-plugin-cc`

## Architecture

```
codesop                     # CLI entrypoint
setup                       # Host integration sync
├── lib/                    # Core shell modules
├── SKILL.md                # /codesop definition
├── commands/               # Slash command files
├── config/
│   └── codesop-router.md   # Router card
├── templates/
│   ├── system/             # System-level AGENTS.md template
│   ├── project/            # PRD.md, README.md templates
│   └── init/               # Init prompt templates
├── tests/                  # Contract tests
│   └── run_all.sh          # Unified test runner
├── AGENTS.md               # → @CLAUDE.md
├── CLAUDE.md               # Project guide
├── PRD.md                  # Living document
```
